# marketing/

Draft posts for promoting the companion repo and the book. Jovan publishes these
himself, by hand. Nothing in this directory is wired to a posting tool or a scheduler,
and nothing here ever auto-posts. A file in this folder is a working draft until the
moment he pastes it into a platform.

## Links and placeholders

- The repo link is filled in every draft: https://github.com/The-825/agent-ops-playbook
- `[BOOK LINK]` stays a placeholder: the book is not launched yet, so no public listing
  exists. Fill it the day the listing goes live (the same trigger as the README link,
  ROADMAP item 3). Until then every draft points at the book with a single pointer
  line. If a draft ever carries `[BOOK LINK]` before the listing exists, cut that line
  rather than point at nothing.

## Files

- `LAUNCH_RUNBOOK.md`: the posting order, spacing, per-channel watch list, and
  comment-response guide. Start here on launch day.
- `reddit-posts.md`: three value-first posts, each matched to a subreddit, with
  posting notes verified against each sub's written rules on 2026-07-21
- `substack-series.md`: the series concept, an eight-post outline, the full first
  post, and its publishing checklist
- `linkedin-posts.md`: three short lesson posts and the cadence plan
- `show-hn.md`: an optional Show HN submission (title plus first comment) and the fit
  warning that matters
- `launch/linkedin-launch-post.md`: the LinkedIn launch announcement for the book (one
  post, first person, posted as written)
- `launch/x-thread.md`: the X launch thread, six posts; post 1 doubles as the
  standalone single-tweet version
- `launch/email-announcement.md`: the plain-text launch email announcement to the
  personal and professional network
- `org-profile-README.md`: paste-ready GitHub org profile for The-825 (destination
  `The-825/.github`, file `profile/README.md`; that repo does not exist yet)

## Writing rules (binding for every draft here)

- First person, Jovan's voice: direct, human, credibility-first, built by an operator.
  Never salesy, never hype.
- Value first. The reader gets the full lesson without clicking anything.
- The repo is linked once per piece. The book gets at most one pointer line: the title,
  "first in a planned series", coming soon. Never summarize book content, never cite
  chapters, never name future titles in the series.
- No pricing, no dates, no launch logistics, anywhere.
- Anecdotes stay at the level of "a production repo I run" / "a system that handles
  regulated PII". No employer detail, no institution names, no client or student
  specifics, ever.
- No em-dashes. Commas, periods, or "·".
- None of the AI-tell vocabulary. Banned outright: `delve`, `journey`, `unique`,
  `passionate`, `testament`, `tapestry`, `multifaceted`, `robust`, `seamlessly`,
  `transformative`, `leverage` as a verb, `game-changer`, `not only ... but also`.
  Also banned: stacked bold-label bullets, "Three things:" scaffolding, and
  rhetorical-question openers.
- Before posting to any subreddit, re-read that community's current sidebar rules. The
  approach notes in `reddit-posts.md` reflect the norms as verified on 2026-07-21;
  norms shift.
- Re-read every draft aloud before posting. If a sentence sounds generated, rewrite it
  or cut it.
