# Mechanical Facts Template

> Part of the companion kit for *From Archivist to Architect* (The Architect's Blueprint, Book 1). Pairs with section 10.6 (The Ledger and the Live Check).

Every hardcoded count in a prose doc rots the day after you write it. Line counts, endpoint counts, flag counts, test counts: they drift silently, readers trust them anyway, and a few stale numbers teach people to distrust the whole doc. This template gives those numbers one home, a MECHANICAL_FACTS.md that a script regenerates deterministically, so prose everywhere else can point at it instead of restating figures that will be wrong by Friday.

## The template

Copy this into your repo root as `MECHANICAL_FACTS.md` and swap the example facts for your own.

````markdown
# MECHANICAL_FACTS.md

Auto-generated mechanical counts for this repo. Trust a number in this file over any
number in prose. Prose docs link here and never restate these figures; if you find a
hardcoded count elsewhere, delete it and point at this file.

Do not hand-edit anything between the START and END markers below. The refresh script
rewrites that region and will overwrite your change without warning.

## Repo-derivable facts

Recomputed from the working tree. Safe to regenerate at any time.

<!-- MECHANICAL_FACTS:START -->
| Fact | Value |
| ---- | ----- |
| Frontend line count (static/app.js) | 4812 |
| API endpoint count (route decorators) | 63 |
| Feature flag count (flag definitions) | 21 |
| Test count (test functions) | 148 |

Generated 2026-01-15 by scripts/refresh_mechanical_facts.py. If this date is more than
a couple of weeks old, rerun the script before quoting anything in this block.
<!-- MECHANICAL_FACTS:END -->

## Point-in-time ledger

Figures that mirror a live system (a production database row count, a deployed config
value) cannot be recomputed from the repo. Record them here as dated, append-only rows.
Never edit or recompute an old row; add a new one.

| Date | Fact | Value | Source |
| ---- | ---- | ----- | ------ |
| 2026-01-15 | Live feature-flag rows | 24 | flags table, production |

Anything with no rebuildable source in the repo and no ledger row is out of scope for
this file: fetch it live from the canonical endpoint and cite your own as-of date.
````

## The two tiers, and why the split matters

The template separates facts into two sections on purpose.

| Tier | Examples | Handling |
| ---- | -------- | -------- |
| Repo-derivable | line counts, file-glob counts, decorator counts, test counts | Regenerated by the script, any time, from the working tree. Never hand-edited. |
| Point-in-time ledger | live table row counts, deployed config values | Appended by a human with a date and a source. Never recomputed, because the repo cannot rebuild them. |

Mixing the two is the classic failure. A live row count pasted into an auto-generated table looks authoritative and rots silently, because no script can refresh it. Keep it in the ledger with its date, and the date itself tells the reader how much to trust it.

## The refresh script

Reference implementation, stdlib only. Edit the `FACTS` list at the top to define your own counts; the rest should not need touching. It rewrites only the region between the markers, so the prose around them is stable, and two runs on the same tree produce identical output apart from the generation date.

```python
"""Refresh the auto-generated block in MECHANICAL_FACTS.md.

Stdlib only. Rewrites only the region between the START and END markers.
Idempotent: two runs on the same working tree produce the same table.

Usage: python3 scripts/refresh_mechanical_facts.py
"""

import datetime
import glob
import re
import sys
from pathlib import Path

TARGET = Path("MECHANICAL_FACTS.md")
START = "<!-- MECHANICAL_FACTS:START -->"
END = "<!-- MECHANICAL_FACTS:END -->"

# Fact definitions. Each entry is (label, kind, spec).
#   kind "lines": spec is a single file path; the fact is its line count.
#   kind "files": spec is a glob; the fact is how many files match.
#   kind "regex": spec is (glob, pattern); the fact is how many lines
#                 across the matching files match the pattern.
FACTS = [
    ("Frontend line count (static/app.js)", "lines", "static/app.js"),
    ("API endpoint count (route decorators)", "regex",
     ("backend/**/*.py", r"@\w+\.route\(")),
    ("Feature flag count (flag definitions)", "regex",
     ("backend/feature_flags.py", r"^FLAG_")),
    ("Test count (test functions)", "regex",
     ("tests/**/*.py", r"^\s*def test_")),
]


def count_lines(path):
    text = Path(path).read_text(encoding="utf-8")
    return len(text.splitlines())


def count_files(pattern):
    return len(glob.glob(pattern, recursive=True))


def count_regex(spec):
    pattern, line_re = spec
    rx = re.compile(line_re)
    total = 0
    for path in sorted(glob.glob(pattern, recursive=True)):
        for line in Path(path).read_text(encoding="utf-8").splitlines():
            if rx.search(line):
                total += 1
    return total


COUNTERS = {"lines": count_lines, "files": count_files, "regex": count_regex}


def build_block():
    rows = ["| Fact | Value |", "| ---- | ----- |"]
    for label, kind, spec in FACTS:
        rows.append(f"| {label} | {COUNTERS[kind](spec)} |")
    today = datetime.date.today().isoformat()
    footer = (
        f"Generated {today} by scripts/refresh_mechanical_facts.py. If this date is "
        "more than\na couple of weeks old, rerun the script before quoting anything "
        "in this block."
    )
    return "\n".join(rows) + "\n\n" + footer


def main():
    text = TARGET.read_text(encoding="utf-8")
    start = text.find(START)
    end = text.find(END)
    if start == -1 or end == -1 or end < start:
        sys.exit(f"error: markers not found or out of order in {TARGET}")
    head = text[: start + len(START)]
    tail = text[end:]
    TARGET.write_text(head + "\n" + build_block() + "\n" + tail, encoding="utf-8")
    print(f"refreshed {TARGET}")


if __name__ == "__main__":
    main()
```

## Wiring it in

Do not rely on anyone remembering to run the script by hand. Put it on the same cadence as whatever doc-sync automation you already run: every N merged PRs, or a weekly cron job in CI that runs the script and commits the diff if the numbers moved. Hand-running is fine for a spot check before you quote a figure, but the cadence is what keeps the file honest when nobody is thinking about it.

Once this file exists, the rule that gives it teeth is a one-liner in your repo's operating doc: prose defers to MECHANICAL_FACTS.md and never restates its numbers. A count that lives in one regenerated place cannot rot in six.
